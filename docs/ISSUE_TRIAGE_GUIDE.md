# Issue triage guide

When triaging a new issue:

1. **Reproduce** — confirm behaviour.
2. **Label** — bug / enhancement / question / security.
3. **Severity** — SEV1/SEV2/SEV3 for bugs; size for enhancements.
4. **Assign** — pick a maintainer or leave open for community.
5. **Milestone** — attach to current release if urgent, else backlog.

Aim to triage within 2 business days.
