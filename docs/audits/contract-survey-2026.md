## Notes

- **Checked arithmetic**: The `No (uses wrapping arithmetic)` flag indicates the contract
  relies on default Rust wrapping semantics for integer arithmetic.  This is a known
  risk area; see Issue #22 and Issue #7 for tracked remediation.
- **Error handling**: Contracts currently using `panic!`-based errors are candidates for
  migration to `Result<T, PuzzleError>` (ADR-0005, Issue #32).
- **Reentrancy**: The `referral` contract performs cross-contract token transfers
  without a reentrancy guard; see Issue #16 for tracked mitigation.
- **Event coverage**: `insurance` and `puzzle_dao` emit no events on state-changing
  operations, hindering off-chain indexing and monitoring.
