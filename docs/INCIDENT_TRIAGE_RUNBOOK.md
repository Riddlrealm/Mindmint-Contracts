# Incident triage runbook

Quick triage when a page arrives.

## First 5 minutes

1. Acknowledge the alert in your pager.
2. Skim the dashboard linked from `docs/RUNBOOK.md`.
3. Determine if this is user-visible vs. internal.
4. If user-visible, open a status banner.

## Next 30 minutes

1. Identify affected contracts / endpoints.
2. Cross-reference `docs/RUNBOOK.md` for the immediate mitigation.
3. Communicate via the on-call channel.
