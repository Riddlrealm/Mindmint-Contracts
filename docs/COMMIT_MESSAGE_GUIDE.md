# Commit message guide

Follow Conventional Commits (already enforced by `.gitmessage`).

## Format

```
<type>(<scope>): <subject>

<body>

<footer>
```

## Types

feat, fix, docs, style, refactor, perf, test, build, ci, chore, revert

## Subject rules

- <= 50 chars.
- Imperative mood ("add", not "added" / "adds").
- No trailing period.

## Body

- Wrap at 72 chars.
- Explain *what* and *why*, not *how*.
